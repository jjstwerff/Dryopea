Lav

A statically-typed scripting language for the Dryopea engine

Lav is an imperative language inspired by Rust but designed to be concise and easy to embed. Every example in this documentation is a live, executable test — the pages below are generated directly from the tests/suite/ source files.

Key characteristics:

  • Variables are declared on first assignment; types are inferred.
  • Functions use implicit return: the last expression in a block is the result.
  • Null is a typed absence value; any field or variable can be null unless marked not null.
  • Strings are UTF-8; len() counts bytes and indexing is byte-based.
  • Collections — vector, sorted, index, hash — share records and can be combined in a single struct.
  • ^ is exponentiation (not bitwise XOR); bitwise operators are &, |, <<, >>.
  • Polymorphic dispatch is supported for struct-enum variants.

Topics

Keywords

Core syntax: if/else expressions, for loops, break/continue, and named breaks for nested loops.

Text

UTF-8 strings: byte-based indexing, slices, concatenation, formatting, and built-in search functions.

Integer

32-bit integers: arithmetic, conversions to/from text and float, bitwise operations, and null from division by zero.

Boolean

Boolean logic: &&/|| and and/or, bitwise &/|/<</>>, and null as the absent/falsy value.

Float

64-bit floating-point: arithmetic, the ^ exponentiation operator, math functions (sin, cos, log, round, ceil, floor), and format specifiers.

Function

Functions: parameter defaults, in-out references with &, early return, and implicit return of the last expression.

Vector

Dynamic arrays: literals, slices, filtered removal with #remove, reverse iteration, and for-loops embedded in format strings.

Struct

Record types: field constraints (limit, not null), computed fields using $, methods, JSON (:j) and pretty-print (:#) formatting.

Enum

Enums: plain value enums with ordering and text conversion; struct enums with field access and polymorphic dispatch.

Sorted

sorted<T[key]>: O(log n) key lookup and in-order iteration in a sorted binary tree; ascending and descending sort keys.

Index

index<T[key]>: balanced red-black tree with strict sorted order, multi-key lookup, and range iteration.

Hash

hash<T[key]>: O(1) average-time key lookup; combining a hash with a vector for ordered traversal and fast access.

File

File system access: creating File handles, listing directory contents, reading file text, and splitting into lines.

Image

Image loading and pixel-level manipulation using the built-in Image type.

Lexer

Built-in lexer library: tokenising source text, handling multi-character operators, comments, string literals, and formatting expressions.

Parser

Built-in parser library: parsing lav source code programmatically from within a lav program.
